최근에 데이터 통신시 json 데이터를 많이 사용한다.
그 이유에 대해서는 검색한번이면 많이들 나오니.. 흔히들 javascript 에서 사용되는 데이터 형식으로 알고 있지만
실제로 javascript 뿐 아니라 서버간의 통신시 json 데이터로 통신하는 경우가 증가하고 있고,
사용해보면 xml 보다는 좋은것(물론 xml만의 장점도 있다) 같다.

json 데이터의 로그를 찍어서 디버깅이나 데이터 확인시 string 이라서 보기가 쉽지가 않다.
특히 복잡한 구조나 데이터가 많을경우는 더 그렇다.

json 데이터 확인을 위해서 쉬운 방법이 없을까 하다가 json 데이터를 html 형식(보기 좋게)으로
보이면 어떻까 하는 마음에 한줄 코딩을 시작 하고 나서..
한번의 검색으로 이미 오픈 소스로 있음을 알게 되었다.

http://json.bloople.net/

잘 만든것 같다. json String 이 아닌 url  의 경우 해당url을 호출한 결과를 파싱해준다.
지금 하는 통합테스트 부분에 넣어도 될 듯 하다.

잘 사용하겠다고 커밋터들에게 메일을 보내고 싶지만 짧은 영어라 그냥 포기한다.
