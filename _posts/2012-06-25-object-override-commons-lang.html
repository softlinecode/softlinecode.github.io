java에서 최상위 객체인 Object method 들을 override 해서 재사용한다.
자주 사용될수도 있고 아니면 사용 할 필요가 없을지도 모른다. 하지만 우리는 간혹 객체를 비교
해당 내용을 보고 싶어한다. 하지만 기본적인 Object 객체의 method를 이용한다면 아마도 많은
실망을 하게 될 것이다

Model 객체가 상속하도록 BaseObject 라는 객체를 만들어서 3개의 method를 재정의 했다.

import java.io.Serializable;
import org.apache.commons.lang.builder.EqualsBuilder;
import org.apache.commons.lang.builder.HashCodeBuilder;
import org.apache.commons.lang.builder.ToStringBuilder;
import org.apache.commons.lang.builder.ToStringStyle;

/**
 * @author softline
 * Object override toString() , equals() , hashCode()
 */
public class BaseObject implements Serializable {
 private static final long serialVersionUID = 3581752944226957057L;
 @Override
 public String toString() {
  return ToStringBuilder.reflectionToString(this, ToStringStyle.MULTI_LINE_STYLE);
 }
 @Override
 public boolean equals(Object o) {
  return EqualsBuilder.reflectionEquals(this, o);
 }
 @Override
 public int hashCode() {
  return HashCodeBuilder.reflectionHashCode(this);
 }
}

이제 Model 객제는 BaseObject를 상속(extends) 해서 사용하게 된다.
그러면 객체의 값 비교나 값에 대한 출력을 가능하게 해준다.

예전에 이러한 기능을 위해서 노가다 (?) 같은 작업을 통한  method를 만들어 봤었는데 지금은
commons 에서 구현해 놓으니 너무 간편해 졌다.

ToStringBuilder.reflectionToString(this, ToStringStyle.MULTI_LINE_STYLE);
2번째 파라미터는 출력에 관한 스타일을 지정할수 있다. 관심 있으면 관련된 문서를 보면 몇가지 유형을 볼수 있을것이다.

ToStringBuilder , EqualsBuilder 를 다른 곳에서도 사용 가능할 것으로 보인다.

자세한 내용은 http://commons.apache.org/lang/ 를 참조하기 바랍니다.
