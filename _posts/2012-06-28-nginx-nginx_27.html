Nginx는 우선 apache 보다는 성능이 좋습니다. 사실 그 동안 apache를 대체할 만한 web server 를 찾기 못하였기때문에 대부분 그냥 무조건적으로 apache를 사용했었는데요.
이제는 Nginx를 검토해도 될듯 합니다. 밑에 단점이라고 명시되어져 있던 레퍼런스나 문서들이 최근에 늘어나고 있고 특히 국내에서도 사용하는 추세 인듯 합니다.


% 장점
  1) Very lightweight.
      .Very low CPU and memory usage.
       .C10K problem: 10,000 inactive HTTP keep-alive connections take about 2.5M memory.
  2) High performance.
       .low latency and scalable on multicore.
  3) Features.
       .Mail & Web reverse proxy server (fast load balancer).
       .Modular Architecture.

% 단점
  1) One main developer – NGINX, Inc. 설립.
  2) No huge community – 점점 확대되고 있음.
  3) Insufficient document(information) – 문서도 마찬가지.
