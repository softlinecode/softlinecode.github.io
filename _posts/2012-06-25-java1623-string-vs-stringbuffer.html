 String.java와 StringBuffer에 대해서 정리를 하면서 StringBuffer 가 빠르다는 결론으로 갈수가
있습니다. 그렇다면 개발자 String을 사용하지 않고 StringBuffer 를 사용해야 하는것인가 에 대한
생각이 들 수 있습니다.

 참고 링크를 보시면 많은 분들이 String 과 StringBuffer 에 대해서 비교 분석을 잘 해놓으셨습니다.
무단으로 링크를 넣어도 문제가 없기를 바라며 제가 따로 비교에 대한 구체적인 테스트 결과를
작성하지는 않겠습니다.

 String 과 StringBuffer에 대한 특징을 잘 파악하고 사용하면 될듯합니다.
대부분 문자열의 경우 생성된 후 변경이 되지 않는 경우(실제로 코드상에는 + 로 붙이고 있어도 compile 후 append 형식으로 변경될 수도 있습니다. 상수처리처럼요) 에는 String을 사용하는게
유리하고 문자열에 대해서 값에 대한 변경이 일어나는 경우 특히 반복문을 통해서 많은 변경횟수가
예상된다면 StringBuffer를 사용하시면 될것 같습니다.

무조건 StringBuffer 가 빠르다거나, String + 연산 사용시 compiler 가 자동으로 append 형식으로
변경해 주기때문에 어느 상황에서나 + 연산을 사용해도 성능에 문제가 없다는 내용은 문제가 있었네요.
