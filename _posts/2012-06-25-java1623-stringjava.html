 개발을 할때 가장 많이 쓰이는게 문자열 처리 인것 같습니다.
 가장 많이 쓰이기에 가장 많은 API 가 있고 언어의 종류와 상관없이 비슷한 API(사실 필수인듯 한)
들이 존재한다.
 이렇게 많이 사용하는 String.java 코드를 한번 정리해 봅니다.

=====================================================================================
public final class String
    implements java.io.Serializable, Comparable<String>, CharSequence
=====================================================================================
String.java Class 를 보면 final 이고 Serializable , Comparable , CharSequence 를 implement 하고
있다. final 은 상수의 표현이고 다른 Interface 에 대해서는 구지 설명하지 않겠습니다.

String.java 에는 변수로

private final char value[];
private final int offset;
private final int count;
private int hash; // Default to 0
private static final long serialVersionUID = -6849794470754667710L;

다섯가지 변수를 가지고 있습니다.
char value[] 를 보면 String 객체 String 값은 char 형 배열로 가지고 있는 것입니다.
int offset 은 시작되는 index를 가리키는 것 같고 , count는 length를 표현 하는것으로 보입니다.
hash 와 serialVersionUID 에 대해서는 설명하지 않겠습니다.

String 의 기본 개념은 char 형의 배열을 가지고 있는 문자열임을 짐작할 수 있습니다.

default 생성자 종류

public String() {
    this.offset = 0;
    this.count = 0;
    this.value = new char[0];
}

public String(String original) {
        int size = original.count;
        char[] originalValue = original.value;
        char[] v;
        if (originalValue.length > size) {
            // The array representing the String is bigger than the new
            // String itself.  Perhaps this constructor is being called
            // in order to trim the baggage, so make a copy of the array.
            int off = original.offset;
            v = Arrays.copyOfRange(originalValue, off, off + size);
        } else {
            // The array representing the String is the same
            // size as the String, so no point in making a copy.
            v = originalValue;
        }
        this.offset = 0;
        this.count = size;
        this.value = v;
}

public String(char value[]) {
        this.offset = 0;
        this.count = value.length;
        this.value = StringValue.from(value);
}

public String(char value[], int offset, int count) {
        if (offset < 0) {
            throw new StringIndexOutOfBoundsException(offset);
        }
        if (count < 0) {
            throw new StringIndexOutOfBoundsException(count);
        }
        // Note: offset or count might be near -1>>>1.
        if (offset > value.length - count) {
            throw new StringIndexOutOfBoundsException(offset + count);
        }
        this.offset = 0;
        this.count = count;
        this.value = Arrays.copyOfRange(value, offset, offset + count);
}

여기까지 default 생성자와 char[] 형식의 문자열을 가지고 생성하는 생성자 입니다.
Arrays.copyOfRange(..) 의 경우 Arrays.java 할때 보겠지만 기본적으로 
public static char[] copyOfRange(char[] original, int from, int to);
char[] 에서 from index 에서 to index까지의 char[]를 copy 하여 return 한다.
결국 this.value 에 새로운 char[] 배열을 할당한다.

2번째 String.java 생성자중 byte[] 을 char[] 형식으로 변환해서 가지게 되는 경우 입니다.
byte[]의 byte 를 StringCoding.decode 를 통해서 char[] 로 변환합니다.

public String(byte bytes[], int offset, int length, String charsetName) throws UnsupportedEncodingException {
        if (charsetName == null)
            throw new NullPointerException("charsetName");
        checkBounds(bytes, offset, length);
        char[] v = StringCoding.decode(charsetName, bytes, offset, length);
        this.offset = 0;
        this.count = v.length;
        this.value = v;
}

public String(byte bytes[], int offset, int length, Charset charset);
public String(byte bytes[], int offset, int length);
public String(byte bytes[]);

-> byte[] 에 offset 과 length 가 문제가 없는지 체크합니다.
StringCoding.decode(charsetName, bytes, offset, length)
->StringCoding.java 에서 정리가 필요하지만 byte[] 를 char[]로 decode 합니다.

String.java 에서 가장 많이 사용되는 equals() 를 보겠습니다.

@Override
    public boolean equals(Object anObject) {
        if (this == anObject) {
            return true;
        }
        if (anObject instanceof String) {
            String anotherString = (String)anObject;
            int n = count;
            if (n == anotherString.count) {
                char v1[] = value;
                char v2[] = anotherString.value;
                int i = offset;
                int j = anotherString.offset;
                while (n-- != 0) {
                    if (v1[i++] != v2[j++])
                        return false;
                }
                return true;
            }
        }
        return false;
    }

1. 우선 String 객체인지 확인합니다.
2. String 객체의 count 값이 동일한지 비교합니다.
3. offset 기준으로 char[] 값을 하나하나 비교합니다.


startsWith , endsWith 에 코드를 보겠습니다.
startsWith의 경우 endsWith도 동일하게 사용하도록 되어져 있습니다.
prefix 의 count 만큼 toffset 에서 prefix의 count 만큼 비교합니다.

public boolean startsWith(String prefix, int toffset) {
        char ta[] = value;
        int to = offset + toffset;
        char pa[] = prefix.value;
        int po = prefix.offset;
        int pc = prefix.count;
        // Note: toffset might be near -1>>>1.
        if ((toffset < 0) || (toffset > count - pc)) {
            return false;
        }
        while (--pc >= 0) {
            if (ta[to++] != pa[po++]) {
                return false;
            }
        }
        return true;
}

public boolean endsWith(String suffix) {
        return startsWith(suffix, count - suffix.count);
}

String 에 hashCode() 에 대해서 알아봅시다.

    /**
     * Returns a hash code for this string. The hash code for a
     * <code>String</code> object is computed as
     * <blockquote><pre>
     * s[0]*31^(n-1) + s[1]*31^(n-2) + ... + s[n-1]
     * </pre></blockquote>
     * using <code>int</code> arithmetic, where <code>s[i]</code> is the
     * <i>i</i>th character of the string, <code>n</code> is the length of
     * the string, and <code>^</code> indicates exponentiation.
     * (The hash value of the empty string is zero.)
     *
     * @return  a hash code value for this object.
     */
    @Override
    public int hashCode() {
        int h = hash;
        int len = count;
        if (h == 0 && len > 0) {
            int off = offset;
            char val[] = value;

            for (int i = 0; i < len; i++) {
                h = 31 * h + val[off++];
            }
            hash = h;
        }
        return h;
    }

주석을 해석해보면 s[0]*31^(n-1) + s[1]*31^(n-2) + ... + s[n-1] 계산된다고 합니다. int 형으로
return 합니다.
hashCode 값은 메모리의 주소값 입니다. 동일한 값에 대해서는 동일한 hashCode가 return 됩니다.
여기서 약간 생각이 들수 있는것은 int 형은 그 한계가 있으므로 다른 문자열인데도 hashCode 가
동일하게 나올수 있는 확률이 있다는 것입니다. 관련해서는 좀더 스펙을 확인해봐야 할 듯 합니다.
