하지만 빌드를 해보면 에러가 발생할 것이다.
해당 에러를 찾다가 잘 설명된 블로그 페이지를 찾게 되었다. 아마도 다른 분들도 나와
비숫한 문제를 경험할것 같아서 해당  링크를 남긴다.
http://blog.openframework.or.kr/archive/20100623

하지만 내경우에 eclipse 에서 maven으로 실행시 문제가 발생하게 되었다. 프롬프트 창에서 실행하면
문제가 없이 빌드가 성공한다.
해당 문제는 내 컴터 path에 잡혀있는 maven과 eclipse 의 maven 버젼이 다른데 아마도 eclipse 에 설치된 버젼이 높은것 같은데 문제가 있나보다 그래서 eclipse 의 maven을 내 컴터에 설치된 maven으로 변경했다.


이렇게 변경하면 된다.

기본적으로 jetty를 이용해서 개발해보려고 한다. tomcat 그동안 많이 써왔으니 한번 다른거 써볼 마음으로 한다. 사실 별만 차이 없을듯 하기는 하다.

프롬프트 명령차
mvn jetty:run-war 

eclipse
Goals : jetty:run-war

이렇게 jettty를 실행해서 localhost:8080 으로 들어가서 appFuse 로그인 화면이 나오면 
로컬에서 새로운 프로젝트 생성에 성공한것이다.

appFuse로 프로젝트 생성시 사실 여러가지 것들이 많이 포함된듯하다.
처음보는 확장자도 있으면 다국적 properties 도 포함되어져 있다. 물론 DB는 MySql를 사용한다.

그래도 약간의 삽질은 들어갔지만 이정도의 프로젝트 구성을 아마 무에서 시작했다면 많은 노력과
더 많은 삽질으 했을것 같다.

우선 해당 프로젝트를 google code svn에 commit을 하였다.
개인적 학습용에 가깝지만 그래도 공개 소스 저장소에 commit 하는 최초의 순간이다.
