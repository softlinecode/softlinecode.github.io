--- layout: post title: Cassandra db(카산드라 DB) 로컬 DB 테스트 date: '2012-06-24T20:40:00.002-07:00' author: Jung Gi Lim tags: - DB - cassandra modified_time: '2012-06-24T20:40:16.922-07:00' blogger_id: tag:blogger.com,1999:blog-2993388283610032812.post-5107118774635198674 blogger_orig_url: http://softline21c.blogspot.com/2012/06/cassandra-db-db-db.html --- Cassandra DB가 설치되었다면 client로 접근해서 한번 데이터를 넣고 조회 해보겠다.

help(?) 명령어로 어떻게 사용하는지 한번 찾아본다.
cassandra> ?
List of all CLI commands:
?                                                                  Same as help.
help                                                          Display this help.
connect <hostname>/<port>                             Connect to thrift service.
describe keyspace <keyspacename>                              Describe keyspace.
exit                                                                   Exit CLI.
quit                                                                   Exit CLI.
show config file                                Display contents of config file.
show cluster name                                          Display cluster name.
show keyspaces                                           Show list of keyspaces.
show api version                                        Show server API version.
get <ksp>.<cf>['<key>']                                  Get a slice of columns.
get <ksp>.<cf>['<key>']['<super>']                   Get a slice of sub columns.
get <ksp>.<cf>['<key>']['<col>']                             Get a column value.
get <ksp>.<cf>['<key>']['<super>']['<col>']              Get a sub column value.
set <ksp>.<cf>['<key>']['<col>'] = '<value>'                       Set a column.
set <ksp>.<cf>['<key>']['<super>']['<col>'] = '<value>'        Set a sub column.
del <ksp>.<cf>['<key>']                                           Delete record.
del <ksp>.<cf>['<key>']['<col>']                                  Delete column.
del <ksp>.<cf>['<key>']['<super>']['<col>']                   Delete sub column.
count <ksp>.<cf>['<key>']                               Count columns in record.
count <ksp>.<cf>['<key>']['<super>']            Count columns in a super column.

명령어가 상당히 심플하다.

우선 데이터를 set 하고 get 해보겠다.
(set,get 이라 정말 프로그램 같다는 생각이 든다.)

cassandra db의 데이터 구조는 따로 설명하지 않겠다. 아마 검색해보면 몇가지 좋은 설명들이 
있을 것이다. 테이블이 없다는 사실에 조금은 적응하기 어려울수도 있지만 한 두번의 데이터를
set , get을 해보면 이해가 쉬울것이라고생각된다.

basicinfo 키워드에 숫자가 붙은 키를 생성해보고 이름의 데이터를 key/value로 넣어보겠다.
(기본 설치가 Keysapce1.Standard1 은 기본적으로 생기는 ksp<Keyspace> , cf<ColumnFamily> 이다. 이게 왜 기본적으로 있는지 궁금하다면 DB 설치 root에 /conf/storage-conf.xml 파일을 보면
알게 된다.

cassandra> set Keyspace1.Standard1 ['basicinfo_1000001'] ['name']= 'limjunggi'
Value inserted.

데이터를 set 하였다. 그럼 다시 해당 데이터를 get 해보자.

cassandra> get Keyspace1.Standard1 ['basicinfo_1000001'] ['name']
=> (column=6e616d65, value=limjunggi, timestamp=1288008491015000)

해당 키에 name을 찾아보니 column 과 value , timestamp 가 나온다.
이번엔 다른 데이터를 set 하면서 age (나이) 까지 넣어 보겠다.

cassandra> set Keyspace1.Standard1 ['basicinfo_1000002'] ['name']= 'bungsun'
Value inserted.
cassandra> set Keyspace1.Standard1 ['basicinfo_1000002'] ['age']= '33'
Value inserted.

두개의 데이터를 set 후 해당 키에 데이터를 get 해보겠다.

cassandra> get Keyspace1.Standard1 ['basicinfo_1000002']
=> (column=6e616d65, value=bungsun, timestamp=1288008590609000)
=> (column=616765, value=33, timestamp=1288008597171000)
Returned 2 results.

name , age 가 나오는것을 볼수 있다.

우선 이것으로 기본적으로 데이터를 넣는 set 방법과 조회하는 get을 데스트해 보았다.

컬럼의 숫자를 조회하는 count 명령어를 테스트 해본다.

cassandra> count Keyspace1.Standard1 ['basicinfo_1000001']
1 columns
cassandra> count Keyspace1.Standard1 ['basicinfo_1000002']
2 columns

해당 키 별로 컬럼의 갯수가 조회가 된다. 해당 키 별 컬럼의 수가 동적이니 해당 컬럼 count를 어떻게 보면 유용하게 사용될수 있을 것 같다.

사용가능한 명령어들이 정리되어서 보인다.

cassandra>

connect                           count Keyspace1.Standard1         count Keyspace1.Standard2
count Keyspace1.StandardByUUID1   count Keyspace1.Super1            count Keyspace1.Super2
del Keyspace1.Standard1           del Keyspace1.Standard2           del Keyspace1.StandardByUUID1
del Keyspace1.Super1              del Keyspace1.Super2              describe keyspace
exit                              get Keyspace1.Standard1           get Keyspace1.Standard2
get Keyspace1.StandardByUUID1     get Keyspace1.Super1              get Keyspace1.Super2
help                              quit                              set Keyspace1.Standard1
set Keyspace1.Standard2           set Keyspace1.StandardByUUID1     set Keyspace1.Super1
set Keyspace1.Super2              show api version                  show cluster name
show config file                  show keyspaces

간단하게 DB 테스트를 했으니 이제 어플리케이션에서 연결하는 작업을 해봐야겠다.
